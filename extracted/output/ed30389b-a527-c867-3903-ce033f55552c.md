# ### Return Value
The FieldBuilder object may be used to add more details to the field. Make sure to set the unit type if the field contains floating-point values. 
# Remarks
The supported types for the keys are Boolean, Byte, Int16, Int32, Int64, ElementId, GUID and String. Floating-point types (Float, Double, XYZ and UV) are not supported because round-off errors frequently cause numeric instability. Subentities are not supported because they require a custom comparison operator. The suggested workarounds are either placing key values into an Array and using array indices for keys, or just using an Array of subentities. Natural comparison of contained types is used for sorting. The supported types for values are the same as for simple fields. See [AddSimpleField(String, Type)](5de0ea30-a58e-4db2-373c-05222a139465.md "AddSimpleField Method") for details. 
Note that a schema may contain a maximum of 256 fields.
